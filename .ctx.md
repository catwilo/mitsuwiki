# mitsuwiki — context
## status: ACTIVE

Personal wiki. Markdown notes, offline browser SPA, terminal search via fzf.
Source of truth: ~/unix-toolkit-tools/mitsuwiki/

## structure
- wiki.sh        — CLI entrypoint (serve|stop|search|open|new|index)
- install.sh     — symlink wiki.sh -> ~/.local/bin/wiki (idempotent)
- index.html     — SPA dark theme, sidebar + marked.js renderer
- marked.min.js  — bundled offline, no CDN
- notes.json     — auto-generated by wiki serve / wiki index (not committed)
- notes/         — all .md notes, organized by category
- assets/        — PDFs and static files referenced from notes

## categories
system | network | audio | video | display | vm | editors | tools | desktop | misc

## commands
  wiki serve [port]    byobu window "mitsuwiki" + python3 http.server + clipso URL
  wiki stop            kill byobu window "mitsuwiki"
  wiki search [query]  fzf over .md contents -> $EDITOR
  wiki open            fzf picker -> $EDITOR
  wiki new <cat/name>  create .md from template -> $EDITOR
  wiki index           rebuild notes.json manually

## integrations
- byobu: server runs in named window "mitsuwiki"
- clipso: URL copied to clipboard on serve
- fzf: search + open picker
- $EDITOR (nvim): all editing

## migration
- source: ~/unix-toolkit-tools/helpcomands/
- text files -> .md per category
- PDFs (ASCII table, sed, ranger, ffmpeg) -> assets/
- WAVs/redundant HTMLs -> discarded
- helpcomands repo -> archive on GitHub after migration complete

- add CDN dependencies (marked.js is bundled)
